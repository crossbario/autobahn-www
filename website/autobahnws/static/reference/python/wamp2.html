WAMP v2¶

WAMP application components are implemented by deriving from the class autobahn.wamp.protocol.ApplicationSession. This class implements the following core interfaces for WAMP application programming:

  • autobahn.wamp.interfaces.ISession
  • autobahn.wamp.interfaces.ICaller
  • autobahn.wamp.interfaces.ICallee
  • autobahn.wamp.interfaces.IPublisher
  • autobahn.wamp.interfaces.ISubscriber

Callers¶

Calling¶

Callers can call remote procedures on other WAMP application components using the following interface:

interface autobahn.wamp.interfaces.ICaller¶

Extends: autobahn.wamp.interfaces.ISession

Interface for WAMP peers implementing role Caller.

call(procedure, *args, **kwargs)¶

Call a remote procedure.

This will return a Deferred/Future, that when resolved, provides the actual result.

If the result is a single positional return value, it’ll be returned “as-is”. If the result contains multiple positional return values or keyword return values, the result is wrapped in an instance of autobahn.wamp.types.CallResult.

If the call fails, the returned Deferred/Future will be rejected with an instance of autobahn.wamp.exception.ApplicationError.

If the Caller and Dealer implementations support cancelling of calls, the call may be canceled by canceling the returned Deferred/Future.

If kwargs contains an options keyword argument that is an instance of autobahn.wamp.types.CallOptions, this will provide specific options for the call to perform.

Parameters:
  • procedure (str) – The URI of the remote procedure to be called, e.g. com.myapp.hello.
  • args (list) – Any positional arguments for the call.
  • kwargs (dict) – Any keyword arguments for the call.
Returns:

obj – A Deferred/Future for the call result - an instance of twisted.internet.defer.Deferred (when running under Twisted) or an instance of asyncio.Future (when running under asyncio).

Call Options¶

To set options when issuing a call, provide a keyword argument options of the following type to autobahn.wamp.interfaces.ICaller.call():

class autobahn.wamp.types.CallOptions(onProgress=None, timeout=None, discloseMe=None, runOn=None)¶

Used to provide options for calling with autobahn.wamp.interfaces.ICaller.call().

__init__(onProgress=None, timeout=None, discloseMe=None, runOn=None)¶

Constructor.

Parameters:
  • onProgress (a callable) – A callback that will be called when the remote endpoint called yields interim call progress results.
  • timeout (float) – Time in seconds after which the call should be automatically cancelled.
  • discloseMe (bool) – Request to disclose the identity of the caller (it’s WAMP session ID) to Callees. Note that a Dealer, depending on Dealer configuration, might reject the request, or might disclose the Callee’s identity without a request to do so.
  • runOn (str) – If present (non-None), indicates a distributed call. Distributed calls allows to run a call issued by a Caller on one or more endpoints implementing the called procedure. Permissible value are: “all”, “any” and “partition”. If runOne == “partition”, then runPartitions MUST be present.

Callees¶

Registering¶

Callees provide endpoints which can be called as remote procedures from other WAMP application components:

interface autobahn.wamp.interfaces.ICallee¶

Extends: autobahn.wamp.interfaces.ISession

Interface for WAMP peers implementing role Callee.

register(endpoint, procedure=None, options=None)¶

Register an endpoint for a procedure to (subsequently) receive calls calling that procedure.

If endpoint is a callable (function, method or object that implements __call__), then procedure must be provided and an instance of twisted.internet.defer.Deferred (when running on Twisted) or an instance of asyncio.Future (when running on asyncio) is returned.

If the registration succeeds the Deferred/Future will resolve to an object that implements autobahn.wamp.interfaces.Registration.

If the registration fails the Deferred/Future will reject with an instance of autobahn.wamp.exception.ApplicationError.

If endpoint is an object, then each of the object’s methods that are decorated with autobahn.wamp.procedure() are registered as procedure endpoints, and a list of Deferreds/Futures is returned that each resolves or rejects as above.

Parameters:
  • endpoint (callable) – The endpoint or endpoint object called under the procedure.
  • procedure (str) – When endpoint is a single event handler, the URI (or URI pattern) of the procedure to register for. When endpoint is an endpoint object, this value is ignored (and should be None).
  • options (An instance of autobahn.wamp.types.RegisterOptions.) – Options for registering.
Returns:

obj – A (list of) Deferred(s)/Future(s) for the registration(s) - instance(s) of twisted.internet.defer.Deferred (when running under Twisted) or instance(s) of asyncio.Future (when running under asyncio).

Registration¶

Each registration of an endpoint at a Dealer is represented by

interface autobahn.wamp.interfaces.IRegistration¶

Represents a registration of an endpoint.

id¶

The WAMP registration ID for this registration.

active¶

Flag indicating if registration is active.

unregister()¶

Unregister this registration that was previously created from autobahn.wamp.interfaces.ICallee.register().

After a registration has been unregistered, calls won’t get routed to the endpoint any more.

This will return a Deferred/Future, that when resolved signals successful unregistration.

If the unregistration fails, the returned Deferred/Future will be rejected with an instance of autobahn.wamp.exception.ApplicationError.

Returns:obj – A Deferred/Future for the unregistration - an instance of twisted.internet.defer.Deferred (when running under Twisted) or an instance of asyncio.Future (when running under asyncio).

Subscribers¶

Subscribing¶

interface autobahn.wamp.interfaces.ISubscriber¶

Extends: autobahn.wamp.interfaces.ISession

Interface for WAMP peers implementing role Subscriber.

subscribe(handler, topic=None, options=None)¶

Subscribe to a topic and subsequently receive events published to that topic.

If handler is a callable (function, method or object that implements __call__), then topic must be provided and an instance of twisted.internet.defer.Deferred (when running on Twisted) or an instance of asyncio.Future (when running on asyncio) is returned.

If the subscription succeeds the Deferred/Future will resolve to an object that implements autobahn.wamp.interfaces.ISubscription.

If the subscription fails the Deferred/Future will reject with an instance of autobahn.wamp.exception.ApplicationError.

If handler is an object, then each of the object’s methods that are decorated with autobahn.wamp.topic() are subscribed as event handlers, and a list of Deferreds/Futures is returned that each resolves or rejects as above.

Parameters:
  • handler (callable or obj) – The event handler or handler object to receive events.
  • topic (str) – When handler is a single event handler, the URI (or URI pattern) of the topic to subscribe to. When handler is an event handler object, this value is ignored (and should be None).
  • options (An instance of autobahn.wamp.types.SubscribeOptions.) – Options for subscribing.
Returns:

obj – A (list of) Deferred(s)/Future(s) for the subscription(s) - instance(s) of twisted.internet.defer.Deferred (when running under Twisted) or instance(s) of asyncio.Future (when running under asyncio).

Publishers¶

Publishing¶

interface autobahn.wamp.interfaces.IPublisher¶

Extends: autobahn.wamp.interfaces.ISession

Interface for WAMP peers implementing role Publisher.

publish(topic, *args, **kwargs)¶

Publish an event to a topic.

If kwargs contains an options keyword argument that is an instance of autobahn.wamp.types.PublishOptions, this will provide specific options for the publish to perform.

If publication acknowledgement is requested via options.acknowledge == True, this function returns a Deferred/Future:

  • if the publication succeeds the Deferred/Future will resolve to an object that implements autobahn.wamp.interfaces.IPublication.
  • if the publication fails the Deferred/Future will reject with an instance of autobahn.wamp.exception.ApplicationError.
Parameters:
  • topic (str) – The URI of the topic to publish to, e.g. com.myapp.mytopic1.
  • args (list) – Arbitrary application payload for the event (positional arguments).
  • kwargs (dict) – Arbitrary application payload for the event (keyword arguments).
Returns:

obj – None for non-acknowledged publications or, for acknowledged publications, an instance of twisted.internet.defer.Deferred (when running under Twisted) or an instance of asyncio.Future (when running under asyncio).
