Detailed Description

WebSocket reader, the receiving leg of a WebSockets connection.

This runs on it's own background thread and posts messages to master thread's message queue for there to be consumed by the application. The only method that needs to be called (from foreground thread) is quit(), which gracefully shuts down the background receiver thread.
